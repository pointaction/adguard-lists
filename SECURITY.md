# Security Policy

## Supported Versions

The latest version of this repository on the `main` branch is supported.

| Version | Supported |
|---------|:---------:|
| Latest (`main`) | ✅ |
| Older commits/releases | ❌ |

## Reporting a Vulnerability

If you believe you have discovered a security issue related to this repository, please **do not** create a public GitHub Issue.

Instead, please report it privately by using one of the following methods:

- GitHub Private Vulnerability Reporting (if enabled)
- GitHub Discussions or Issues for **non-security** problems only

When reporting a security issue, please include:

- A clear description of the issue
- Steps to reproduce (if applicable)
- The affected file(s) or list(s)
- Any relevant screenshots or logs

## Scope

This repository contains DNS blocklists, allowlists, and filter lists for DNS filtering solutions such as:

- AdGuard Home
- Technitium DNS Server
- Compatible DNS filtering software

Security reports should relate to the repository itself, such as:

- Malicious or compromised domains accidentally whitelisted
- Legitimate domains incorrectly replaced by malicious ones
- Supply chain concerns involving repository content

False positives, false negatives, or requests to add or remove domains are **not security vulnerabilities** and should be submitted as normal GitHub Issues.

## Response

While response times cannot be guaranteed, security reports will be reviewed as soon as reasonably possible. Valid issues will be investigated and corrected in a future update.

Thank you for helping improve the security and quality of this project.
